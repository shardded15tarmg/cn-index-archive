# cn-index-archive

本仓库用于归档和发布多个独立 HTML 页面。内容以静态页面形式存放，不针对任何特定域名、网站或服务。仓库内页面可作为独立资源访问或嵌入使用。

## 目录说明

```
.
├── README.md          # 本文件
├── index.html         # 入口页面（可选）
├── pages/             # 存放归档页面
│   ├── page-001.html
│   ├── page-002.html
│   └── ...
└── assets/            # 页面依赖资源（CSS、JS、图片等）
    ├── style.css
    └── script.js
```

- `pages/` 目录为页面归档主体，每个 HTML 文件为一个独立归档单元。
- `assets/` 目录存放页面使用的共用或独立资源文件。
- 根目录下的 `index.html` 可作为导航或说明页面（视需要创建）。

## 页面归档说明

- 归档页面均为独立 HTML 文件，可直接在浏览器中打开。
- 页面之间无相互依赖关系，可单独部署或引用。
- 页面内容遵循通用 Web 标准，不依赖外部服务。
- 文件名按顺序命名（如 `page-001.html`），便于索引和管理。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面会通过 commit 记录。
- 欢迎提交 Issue 或 Pull Request 提出改进建议或补充页面。
- 提交新页面时，请确保：
  - 页面为独立完整 HTML 文件
  - 不包含恶意代码或违反通用政策的内容
  - 资源文件放入 `assets/` 对应子目录
- 历史版本可通过 Git 提交记录追溯。

## 使用方式

1. 直接访问 GitHub Pages 部署地址（如已启用）查看在线版本。
2. 克隆仓库到本地，用浏览器打开 `pages/` 下任一 HTML 文件。
3. 可将单个页面嵌入到其他应用中，注意保持资源路径正确。
